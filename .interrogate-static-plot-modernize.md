# Destination: gps_plot static-plotting modernization (Path B)

## Problem
gps_plot's matplotlib static production (the brunnur `{STA}-plate-{90d,year}` PNGs)
is slow and export-incomplete. The **semantic red/green header** (color flags status
from the last data point) renders only under the EPS/`ps` backend today — PNG is an
ImageMagick `convert` of the EPS, PDF gets no colored header — and the dominant cost
is `usetex` being reset and re-run **per figure**. gps_plot stays the **static**
foundation; dynamic/interactive output is aflogun's lane.

## In scope (`timesmatplt.py`, Path B)
- Native single-`savefig` export to **PDF / PNG / EPS** (drop the `convert` subprocess
  and the `save_allow=[eps,pdf]` restriction; the "EPS-only header" is a code artifact —
  `usetex` renders on all backends).
- **Preserve the data-driven red/green semantic header identically in all three formats.**
- Strip the per-plot `usetex`/rcParams reset (`:245-256`); set style once; reuse figures
  where safe (`:260`); avoid redundant data re-reads (`:147`) and the extra `bbox_inches`
  render (`:498`).
- *(Secondary)* a figure-returning seam: `plot` builds+returns a `Figure`, saving is
  separate — enabler for the "iPython-like" REPL loop from nvim+terminal.

## Out of scope
- Frozen legacy snapshots (`gasmatplt_workingon15May17.py`, `…_bgo_15May17.py`) — do-not-delete.
- GMT/pygmt (`gmtplot.py`); Path A / `highlight_text` (deferred — revisit only if
  measurement demands); dynamic/interactive output (aflogun); repo-wide typing/lint
  overhaul (only touched code → `dev_viz` standard); `gasmatplt.py` (fast-follow after
  timesmatplt proves the pattern).

## Definition of done
- A station renders **visually identical** incl. the **correct red/green header** in
  **PDF, PNG, EPS**, each via one direct `savefig`; header color matches the
  last-data-point logic **in each format**.
- Documented **before/after wall-clock** on 10–20 stations, with the **TeX cost isolated**
  (data-backs the later Path-A decision).
- Hot spots removed (no per-plot `usetex` reset, no `convert`, style-once, figure reuse).
- Touched code ruff/black/mypy-clean + a test asserting 3-format export + header-color logic.
- **Acceptance = BGÓ by-eye on a sample** (exact cross-backend pixel match is unrealistic).

## Hard constraints
- Red/green header is **semantic**, not decorative — preserve exactly, all formats.
- Path B keeps `usetex` → assumes a working TeX install on target hosts.
- **Human checkpoint:** Fable stops at "samples + timings ready for sign-off"; does not
  self-declare done, does not commit. BGÓ eyeballs, then commit.
